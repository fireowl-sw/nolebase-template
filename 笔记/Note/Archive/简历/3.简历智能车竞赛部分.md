## 面试可能遇到的问题与答案

### 1. 赛道识别算法原理

**Q: 如何通过电磁传感器识别不同的赛道类型？**
A: 采用差比和偏差算法。四路传感器左右各两路，计算 `CR = (AR_RIGHT - AR_LEFT) / 10`。通过 CR 值判断车辆相对于赛道中心线的位置：
- CR 在 [-23, 40] 范围内持续较长时间 → 长直道
- CR 超出 [-85, 90] 范围 → 大弯道
- 通过累计计数器（ChuteCtn、WANCtn）达到阈值后自动切换赛道状态

**Q: 为什么要设计 6 种赛道状态，而不是直接用 3 种？**
A: 因为赛道之间存在过渡状态。例如"普通到弯"和"长直道到弯"需要不同的 PID 参数来平滑过渡，避免切换时车辆抖动。此外还有"偏离黑线"的异常情况需要特殊处理。

**Q: 3 种宏观赛道特征和 6 种细分工况有什么区别？**
A: 这是两个不同维度的状态划分。**3 种宏观赛道特征**（普通弯道、长直道、大弯道）是根据当前传感器数据自动识别的"我在哪"，通过累计计数器和滞后判断动态切换。**6 种细分工况**（普通道中间、长直道、普通到弯、直到弯、弯内、偏离黑线）是基于"当前赛道 + 上一段赛道"的组合状态，用于查表选择具体 PID 参数。核心区别在于：宏观特征决定控制策略，细分工况选择具体参数，考虑历史状态可以平滑过渡。

---

### 2. PID 控制设计

**Q: 舵机控制和电机控制有什么区别？为什么舵机只用 PD？**
A: 舵机控制位置，需要快速响应，积分项会引入相位滞后导致转向迟缓，所以只用 PD。电机控制速度，需要消除稳态误差，必须加 I 项。舵机用位置式 PD（`f = 3800 + rkp*CR + rkd*(CR-2*preCR+ppreCR)`），电机用增量式 PID（`speed += mkp*Δerror + mki*error + mkd*Δ²error`）。

**Q: 为什么微分项要用二阶差分 `CR-2*preCR+ppreCR` 而不是简单的一阶差分？**
A: 二阶差分相当于 `(CR-preCR) - (preCR-ppreCR)`，即误差变化率的变化率（加速度）。相比一阶差分能更有效抑制高频噪声，避免舵机在高频干扰下抖动，转向更平滑。

---

### 3. 参数调试方法

**Q: 拨码开关的作用是什么？为什么不直接在代码里改参数？**
A: 拨码开关可以在不重新烧录代码的情况下快速切换 4 组预设参数方案，方便在赛道边进行对比测试。先确定最优方案后再固化到代码中，极大提高了调参效率。

**Q: 如何确定 PID 参数的初始值？调试流程是什么？**
A: 先调 P 参数：从小到大增加，直到响应足够快但开始振荡；然后加 D 参数抑制振荡；最后加少量 I 参数消除稳态误差。针对不同赛道状态分别调试，记录最优值后存入参数表。实际调参时采用"先普通道、后弯道、最后过渡状态"的顺序。

---

### 4. 系统架构与性能

**Q: 为什么选择 10ms 的控制周期？**
A: 10ms 是在控制精度和系统负载之间的平衡。智能车速度约 2m/s，10ms 内行驶约 2cm，足以响应赛道变化。同时 10ms 周期下 CPU 占用率约 60%，留有余量处理其他任务。

**Q: 如何保证控制系统的实时性？**
A: 采用 RTI 中断实现 10ms 精确定时，主循环采用轮询方式等待 AD 转换完成。关键控制逻辑在中断外执行，避免中断嵌套导致的时序混乱。PLL 超频到 40MHz 提供足够算力。

**Q: 速度控制误差 < 5% 是如何达到的？**
A: 通过编码器测速形成闭环，采用增量式 PID 消除稳态误差。针对不同赛道状态预设不同的目标速度（直道 75-80、弯道 42），并在偏离黑线时降速以保证安全。实测误差主要来源于编码器分辨率和机械传动间隙。

---

### 5. 团队与项目管理

**Q: 作为控制算法负责人，如何与硬件团队协作？**
A: 需要明确传感器特性（如电磁传感器输出范围、响应延迟）、电机和舵机的参数（如 PWM 频率、死区）。硬件选型时提出控制精度需求，调试时共同定位问题（如信号噪声是否来自硬件滤波不足）。

**Q: 遇到的最大技术挑战是什么？如何解决的？**
A: 最大挑战是赛道状态误判导致车辆失控。例如在弯道入口处 CR 值波动大，容易错误切换到直道模式。解决方案是引入累计计数器和滞后判断（需要连续 N 个周期满足条件才切换），并增加速度标志 `Flag_gaosu` 作为辅助判断依据。

**Q: 项目中你最有成就感的一件事是什么？**
A: 是看到智能车第一次成功跑完全程不冲出赛道。之前一直存在的问题是弯道切换时车辆抖动严重，通过反复调试发现是参数切换过于 abrupt。后来设计了过渡状态和渐进式参数切换，最终实现了平滑过渡。那一刻验证了自己设计的控制算法是可行的。

void Roadjudge(void){

  if(Flag_Chute==0)         //普通弯道情况 0

  {  GeneralCtn=0;

     CurveCtn=0;

     if(CR>=-23&&CR<=40)        //此时对应一个车轮的内侧压线       @@@@@@@@@@@@@@@@@@@@@@@@@@@@

       {

        ChuteCtn++;

        WANCtn=0;

      }

     if(CR<-23||CR>40) {                 //                            @@@@@@@@@@@@@@@@@@@@@@@@@@

      if(CR<-85||CR>90){                 //对应车轮的外侧压线        @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@

        ChuteCtn=0;

        WANCtn++;

      }

     else {

      ChuteCtn=0;

      WANCtn=0;

     }

     }

     if(ChuteCtn>10000)        // 300

       {

         Flag_Chute=1;            

       }

     if(WANCtn<-10000){

      Flag_Chute=2;

     }

  }

 if(Flag_Chute==1)            //长直道情况   1

 {

   ChuteCtn=0;

   GeneralCtn=0;

   WANCtn=0;

   if(CR>40)CurveCtn++;         //     60               @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@

   if(CR<-23)CurveCtn--;         //   -60               @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@

     if(CurveCtn>1||CurveCtn<-1)

       {

         if(Flag_gaosu==1)             //高速时转入大弯道情况

         {     

            Flag_Chute=2;

         }

         if(Flag_gaosu==0)

            Flag_Chute=0;         //低速时转入普通弯道情况

       }

 }  

if(Flag_Chute==2)            //大弯道情况   2

  {

    ChuteCtn=0;

    CurveCtn=0;

    WANCtn=0;                                      //   @@@@@@@@@@@@@@@@@@@@@@@

    if(CR>-23&&CR<35)GeneralCtn++;

    else GeneralCtn=0;

    if(GeneralCtn>1300)Flag_Chute=0;

    //if(GeneralCtn>130)Flag_Ct=1;

    //else Flag_Ct=0;

    //if(k_abs(Turn_C-Turn<54)

    //   {

    //    Flag_Zhj=1;

    //  }

    // else

    //   {

    //  Flag_Zhj=0;

    //  }

    //   if(Flag_Pwm==1&&Flag_Ct==1)Flag_Chute=1;   

       }

}

void rudder_ctrl(void){  

  if(Flag_Chute==0)  //普通道

    {

      Flag_Pwm=0;

      if(CR<40&&CR>-23){

      rkp=b_rkp[0];

      rkd=b_rkd[0];

      }

      else{

        rkp=b_rkp[6];

        rkd=b_rkp[6];

      }

    }

       else if(Flag_Chute==1) //长直道

       {

          Flag_Pwm=1;

          if(Flag_gaosu==0)        //disu

            {

              rkp=b_rkp[1];     //5

              rkd=b_rkd[1];

             }                   

           else                   //gaosu

             {

               rkp=b_rkp[2];       //4

               rkd=b_rkd[2];

             }

          }

        else if(Flag_Chute==2)   //大弯道

         {

         if(Flag_Pwm==0)        //普通道到大弯道

           {

             rkp=b_rkp[3];

             rkd=b_rkd[3];

           }

         else if(Flag_Pwm==1)       //直道  到  大弯道

          {

             rkp=b_rkp[4];

             rkd=b_rkd[4];

          }

          else{

             rkp=b_rkp[5];

             rkd=b_rkd[5];

          }

         }

           f=3800+rkp*CR+rkd*(CR-2*preCR+ppreCR); //舵机的PID算式

           ppreCR=preCR;//计算之后向前推进赋值 为下次计算做准备

           preCR=CR;           

         }

 void motor_ctrl2(void){

    int error,m_perror,m_ierror,m_derror;

    int pre_error=0;

    int ppre_error=0;

    error=ideal_speed-Pulse_count;

    m_perror = error - pre_error;

    m_ierror=error;

    m_derror=error-2*pre_error+ppre_error;

    ppre_error=pre_error;

    pre_error=error;

    speed+=mkp*m_perror + mki*m_ierror + mkd*m_derror;  //速度PID控制算式

   if(speed>=9000)  speed=9000;

   if(speed<=-4000) speed=-4000;

    if(speed>=0)

      {

        PWMDTY45=0;

        PWMDTY01=(int)speed/10;

    }

    else

    {

        PWMDTY45=(int)(-speed)/10;

        PWMDTY01=0;

    }

 }
